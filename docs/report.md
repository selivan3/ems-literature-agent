# Отчёт по агенту поиска научных статей

## Цель

Создать агента, который помогает быстро собрать научную базу по узкой теме электромембранных систем: как диссоциация и рекомбинация молекул воды влияют на процессы переноса.

## Почему выбран формат плагина

Формат плагина удобен для учебного задания: он отделяет роль агента, навык, промпт и внешние инструменты. В результате агент не является просто текстовой инструкцией. Его можно подключить к среде, где доступен MCP, и использовать как повторяемый инструмент.

## Что подключено

В корне репозитория находится `.mcp.json`. В нём настроен сервер Firecrawl MCP:

- поиск открытых страниц;
- извлечение содержимого страниц;
- проверка страниц издателей;
- дополнительный поиск по eLIBRARY и страницам научных организаций, когда нет удобного открытого API.

## Логика агента

1. Сначала агент расширяет поисковую формулу.
2. Затем ищет статьи в обязательных базах.
3. После этого проверяет открытые источники через Firecrawl MCP.
4. Потом удаляет дубли по DOI и названию.
5. В конце делает русскоязычный отчёт с таблицей, резюме и научным анализом.

## Что считается хорошим результатом

Хороший результат: не максимальное количество ссылок, а проверяемая подборка работ, где понятно:

- какая статья теоретическая;
- какая статья экспериментальная;
- где есть DOI;
- где есть полный текст;
- какие выводы реально относятся к диссоциации и рекомбинации воды;
- какие работы нужно читать первыми.

## Ограничения

Scopus и Web of Science часто закрыты за институциональным доступом. Поэтому агент честно разделяет:

- найденные и проверенные записи;
- записи без полного текста;
- открытые статьи;
- источники, которые требуют ручной проверки.

## Вывод

Агент решает практическую задачу спецсеминара: превращает разрозненный поиск по научным базам в воспроизводимый исследовательский процесс с промптом, MCP-интеграцией и отчётом.
