# Отчёт по агенту поиска научных статей

## Цель

Создать агента, который помогает быстро собрать научную базу по узкой теме электромембранных систем: как диссоциация и рекомбинация молекул воды влияют на процессы переноса.

## Формат

Формат плагина отделяет роль агента, навык, промпт и внешние инструменты. В результате агент выглядит не как разовая инструкция, а как повторяемый исследовательский инструмент, который можно подключить к среде с MCP.

## Что подключено

В корне репозитория находится `.mcp.json`. В нём настроен сервер Firecrawl MCP:

- поиск открытых страниц;
- извлечение содержимого страниц;
- проверка страниц издателей;
- дополнительный поиск по eLIBRARY и страницам научных организаций, когда нет удобного открытого API.

Отдельно добавлен слой навыков Firecrawl:

- `firecrawl-search` — поиск страниц-кандидатов по научной формуле;
- `firecrawl-scrape` — извлечение содержимого известных страниц статей, препринтов и журналов;
- `firecrawl-map` — поиск нужных URL внутри сайта журнала, выпуска или лаборатории;
- `firecrawl-crawl` — ограниченный обход небольшого раздела сайта;
- `firecrawl-interact` — работа с динамическими страницами, формами и пагинацией;
- `firecrawl-parse` — разбор законно полученных PDF, HTML и выгрузок;
- `firecrawl-agent` — узкое структурированное извлечение из открытых страниц.

## Логика агента

1. Сначала агент расширяет поисковую формулу.
2. Затем ищет статьи в обязательных базах.
3. После этого проверяет открытые источники через Firecrawl MCP.
4. Затем применяет навыки Firecrawl: ищет страницы-кандидаты, извлекает страницы статей, строит карту сайта, при необходимости ограниченно обходит раздел, проходит динамические страницы, разбирает законно полученные файлы и проверяет связанные публикации через дополнительные целевые запросы.
5. Потом удаляет дубли по DOI и названию.
6. В конце делает русскоязычный отчёт с таблицей, резюме и научным анализом.

## Что считается хорошим результатом

Хороший результат: не максимальное количество ссылок, а проверяемая подборка работ, где понятно:

- какая статья теоретическая;
- какая статья экспериментальная;
- где есть DOI;
- где есть полный текст;
- какие выводы реально относятся к диссоциации и рекомбинации воды;
- какие работы нужно читать первыми;
- какие записи подтверждены открытым веб-извлечением, а какие требуют ручной проверки в Scopus, Web of Science или eLIBRARY.

## Ограничения

Scopus и Web of Science часто закрыты за институциональным доступом. Поэтому агент честно разделяет:

- найденные и проверенные записи;
- записи без полного текста;
- открытые статьи;
- источники, которые требуют ручной проверки.

Firecrawl не используется для обхода закрытого доступа. Если полный текст закрыт, агент сохраняет страницу статьи, DOI и причину ограничения, но не выдаёт закрытую статью за скачанную. Широкий обход сайтов издательств запрещён: у обхода должны быть домен, раздел, глубина, лимит страниц и понятная исследовательская цель.

## Вывод

В результате получается воспроизводимый исследовательский конвейер: формула запроса, сбор карточек статей, проверка DOI, анализ релевантности и итоговый отчёт по научной литературе.
