# Навык скрейпинга Firecrawl

Используй этот навык как второй слой после поиска в научных базах. Он нужен не для обхода закрытого доступа, а для аккуратного сбора открытых страниц статей, аннотаций, DOI, ссылок на полные тексты и связанных публикаций.

## Когда включать навык

- есть поисковая формула, но ещё нет списка страниц;
- найдена страница статьи, журнала, архива или университета;
- нужно проверить DOI, аннотацию, список авторов или ссылку на открытый полный текст;
- нужно пройти по связанным страницам выпуска, списка литературы, препринта или профиля публикации;
- нужно разобрать локально сохранённый PDF, HTML или выгрузку из базы.

## Рабочие навыки Firecrawl MCP/CLI

| Навык | Задача в агенте |
| --- | --- |
| `firecrawl-search` | превращает поисковую формулу в список страниц-кандидатов |
| `firecrawl-scrape` | извлекает содержимое известной страницы статьи, журнала или препринта |
| `firecrawl-map` | находит нужные URL внутри сайта журнала, выпуска, архива или лаборатории |
| `firecrawl-crawl` | ограниченно обходит небольшой раздел сайта, если нужна серия страниц |
| `firecrawl-interact` | помогает работать со страницами, где нужны переходы, формы, пагинация или раскрытие блоков |
| `firecrawl-parse` | разбирает локальные PDF, HTML и выгрузки, если они получены законно |
| `firecrawl-agent` | выполняет узкое структурированное извлечение из открытых страниц |
| `firecrawl-research-index` | ищет связанные научные работы, цитирующие статьи, источники из списков литературы и близкие публикации |
| `firecrawl-build-onboarding` | проверяет, что ключ `FIRECRAWL_API_KEY` подключён через окружение, а не хранится в репозитории |
| `firecrawl-build-*` | используется только для встраивания Firecrawl в код агента и выбора правильной конечной точки |

## Рабочий конвейер

1. Сформируй русские и английские запросы.
2. Найди первичный список страниц через `firecrawl-search`.
3. Если известен сайт, но не известна точная страница, используй `firecrawl-map`.
4. Для каждой страницы статьи выполни извлечение через `firecrawl-scrape`.
5. Если нужно несколько страниц одного раздела, используй `firecrawl-crawl` с ограничением по разделу, глубине и числу страниц.
6. Если страница неполная из-за динамической разметки, пагинации или формы поиска, переходи к `firecrawl-interact`.
7. Если есть законно полученный локальный PDF, HTML или выгрузка, разбери его через `firecrawl-parse`.
8. Если требуется структурированная таблица из открытых страниц, используй `firecrawl-agent` только с узкой задачей и ограничением по источникам.
9. Для сильных найденных работ расширь подборку через `firecrawl-research-index`: похожие статьи, цитирующие работы, источники из списка литературы.
10. Удали дубли по DOI, названию и авторам.
11. В карточке статьи отдельно укажи, что подтверждено веб-извлечением, а что требует ручной проверки в Scopus, Web of Science или eLIBRARY.

## Правила безопасности

- Не сохраняй `FIRECRAWL_API_KEY`, пароли и токены в репозитории.
- Не пытайся обходить платный доступ к статьям.
- Не запускай широкий `firecrawl-crawl` по сайту издательства без лимитов.
- Не давай `firecrawl-agent` широкую задачу вроде `найди всё по теме`; сначала задай источники, поля и предел расходов.
- Не разбирай через `firecrawl-parse` частные или сомнительно полученные файлы.
- Не считай страницу издателя полноценной заменой записи в Scopus или Web of Science.
- Не выдавай аннотацию или похожую статью за полный текст.
- Если Firecrawl нашёл страницу, но не нашёл DOI, пиши `DOI не найден на открытой странице; нужна ручная проверка`.
- Если полный текст не найден, пиши `полный текст не найден` или `полный текст закрыт`, а не придумывай доступ.

## Итоговые поля

После применения навыка у каждой работы должны быть:

- источник обнаружения;
- страница статьи;
- DOI или явная пометка `не найдено`;
- ссылка на открытый полный текст, если он доступен законно;
- извлечённая аннотация;
- краткое русское резюме;
- пометка о качестве проверки: `проверено`, `нужна ручная проверка`, `полный текст закрыт`.
